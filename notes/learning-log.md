# Learning Log

## Repository
A repo is basically the home of a project and it stores all the files
and keeps the full history of every change.

## Issues
Issues are like a to-do list inside GitHub. Each task or bug becomes
an issue so the team knows what needs to be done. I made three issues
for this assignment.

## Branches
Instead of editing the main code directly, you make a branch — a safe
copy where you can change things without breaking main. This made sense
to me once I actually did it.

## Pull Requests
A PR is how you ask for your branch changes to be merged into main.
The cool part was linking it to an issue with "Closes #1" when I
merged the PR, the issue closed automatically. That showed me how
the work and the task are connected.

## Forks
A fork is your own copy of someone else's repository, so you can
experiment without touching the original.

## GitHub Projects
A Projects board is a visual tracker (Todo / In Progress / Done).
Issues are individual tasks, but the board shows the status of
everything in one place like a Kanban board built into GitHub.
