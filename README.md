## What I Learned About GitHub

   - **Repository:** A project's home - stores all files and their full history.
   - **Issue:** A way to track tasks, bugs, or ideas to work on.
   - **Branch:** A separate copy of the code where you can make changes safely without affecting main.
   - **Pull Request:** A request to merge changes from one branch into another, where others can review first.
   - **Fork:** Your own copy of someone else's repository.
   - **GitHub Projects:** A board (like Kanban) to organize and track issues and tasks visually.
